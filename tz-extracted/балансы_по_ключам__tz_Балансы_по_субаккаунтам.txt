Назначение
Показать текущий суммарный баланс компании и разложить его по API-ключам (счетам/трейдерам) в наглядном виде.
Что берём из источника
Для каждого API-ключа запрашиваем текущий баланс счёта. (эндпойнт гет воллет баланс)
Подготовка данных
Собираем список активных API-ключей.
Получаем по каждому значение баланса.
Формируем таблицу «ключ — баланс».
Расчёты и правила отображения
«Баланс по ключу» — число в деньгах для каждого ключа.
Нулевые и заблокированные ключи можно:
скрывать по умолчанию,
или показывать в конце списка — выбрать и зафиксировать одно поведение.
Визуализация
Диаграмма «Баланс по API-ключам»
Горизонтальная столбиковая диаграмма.
По вертикали — список ключей (имя трейдера и ID ключа).
По горизонтали — шкала денег.
Каждый столбик — зелёный, длина пропорциональна величине баланса.
На столбике — подпись с числом в деньгах (например, «81 079,00 $»).
Подсказка по наведению: имя ключа, точное значение баланса, доля от общего баланса в процентах.
Сортировка и управление
По умолчанию сортировать по убыванию баланса.
Переключатели сортировки: по возрастанию, по имени ключа.
Поиск по имени/ID ключа.
Опция «Показать только топ-N» (например, 10) и кнопка «Показать все».
Формат чисел
Разделители тысяч, два знака после запятой.
