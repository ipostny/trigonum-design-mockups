Назначение
Показать, сколько сделок за выбранный период совершено каждым API-ключом (счётом/портфелем). Визуализация — горизонтальные столбики с числом сделок.
Источник данных
Берём сделки из транзакшн-лога.
Каждая сделка привязана к субаккаунту
Период задаётся в настройках (неделя, месяц, произвольный диапазон).
Часовой пояс — интерфейсный.
Подготовка данных
Загружаем все сделки за выбранный период.
Учитываем только завершённые сделки (есть факт исполнения).
Если есть исправления по одной и той же сделке, считаем один раз — по последней версии.
Расчёт
Для каждого API-ключа считаем количество сделок.
Итоговая таблица: «API-ключ — количество сделок».
Ключи без сделок в периоде не показываем.
Визуализация
Горизонтальная столбиковая диаграмма.
По вертикали — список API-ключей (название и идентификатор).
По горизонтали — шкала количества сделок.
Каждый столбик окрашен в один оттенок зелёного.
Число сделок выводим внутри правой части столбика или над его концом (например, «37»).
Подсказка при наведении: имя ключа и точное количество сделок.
Сортировка и управление
По умолчанию сортировать по убыванию количества сделок.
Переключение периода — общее с остальными индикаторами.
Краевые случаи
Если по всем ключам ноль сделок — показываем пустую диаграмму с подписью «Нет сделок».
Если ключей очень много — показываем топ-N (например, 10) и кнопку «Показать все».
При равенстве значений у нескольких ключей порядок — по названию.
