# Чеклист проверки мокапов TrigonumTrade

## Инструкция для Claude

Ты - QA-аналитик. Тебе нужно проверить HTML-мокапы на соответствие требованиям из ТЗ и эталонным скриншотам.

### Порядок работы для КАЖДОГО раздела:

1. **Открой страницу эталонов:** https://ipostny.github.io/trigonum-design-mockups/reference.html
2. Найди нужный раздел, изучи скриншот из Notion (левая колонка) и текст ТЗ (правая колонка)
3. Нажми на кнопку "Открыть мокап" чтобы перейти к мокапу
4. Прокликай все интерактивные элементы мокапа (вкладки, кнопки, раскрывающиеся строки)
5. Сравни каждое требование из ТЗ с тем, что видишь в мокапе
6. Также сравни визуально скриншот-эталон с мокапом
7. Запиши результат в формате ниже

### Формат отчёта (для каждого раздела):

```
## [Название раздела]
Мокап: [URL]
Эталон: reference.html#[section_id]

### Соответствие ТЗ:
- [ ] Требование 1: [PASS/FAIL/PARTIAL] - комментарий
- [ ] Требование 2: ...

### Визуальные расхождения со скриншотом:
- ...

### Общий статус: [PASS / PARTIAL / FAIL]
### Рекомендации по исправлению:
- ...
```

## Раздел 1: Тим Лид -- Аналитика (Команда)
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#01_tl_analytics
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/team-analytics.html

### Требования из ТЗ для проверки:
1. Верхний блок "Сводка": карточки "Капитал под управлением", "Изменение за день", "Изменение за неделю"
2. График изменения капитала за последнюю неделю
3. Таблица 1 "Команда" с колонками (строго в этом порядке): Старший трейдер, Капитал под управлением, Unrealized PnL, Realized PnL за день, Realized PnL за неделю, PM команды
4. Клик по строке Старшего трейдера раскрывает Таблицу 2 "Трейдеры" (дропдаун)
5. Таблица 2 "Трейдеры" колонки: Трейдер, Капитал, Unrealized PnL, Realized PnL за день, Realized PnL за неделю, Количество сделок, Max DrawDown за неделю
6. Кнопки "Раскрыть все" / "Свернуть все"
7. Сортировка по любой колонке в обеих таблицах
8. Фильтры: Период (День/Неделя/Месяц/Квартал/Год)
9. Условное раскрашивание PnL: градиент красный-нейтральный-зелёный
10. Клик по имени PM/трейдера -- ссылка на ЛК (новая вкладка)

## Раздел 2: Тим Лид -- Торговля
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#02_tl_trading
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/trading-terminal.html
**Также:** https://ipostny.github.io/trigonum-design-mockups/role-workspaces.html

### Требования из ТЗ для проверки:
1. Две основные вкладки верхней навигации: Аналитика | Торговля
2. Активная вкладка визуально подсвечена
3. Внутри "Торговля" два под-таба: Торговля (основной) и Аналитика
4. Внутри "Аналитика" под-табы категорий: Базовая информация (CMP), Long/Short, Индекс страха/жадности, Новости, Карта ликвидаций
5. Переключение мгновенное через клик
6. Активный под-таб визуально выделен
7. Состояния: по умолчанию / наведён / активен / недоступен

## Раздел 3: Тим Лид -- Результативность
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#03_tl_performance
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/team-analytics.html

### Требования из ТЗ для проверки:
**Подвкладка "Аналитика команды":**
1. Переключатель периода: День / Неделя / Месяц / Квартал / Год / Custom
2. Кнопки "Раскрыть все" и "Свернуть все"
3. Карточки метрик: Капитал под управлением (крупное число с разделителями тысяч), Изменение за день (абсолют + %, зелёный/красный)
4. Линейный график "Изменение капитала за последнюю неделю" (7 точек, Пн-Вс, подсказка при наведении)
5. Иерархическая таблица "Команда": строки Старшего трейдера сворачиваются/разворачиваются
6. Строка "Старший трейдер": аватар/иконка, ФИО, роль, Капитал (суммарно), Unrealized PnL, Realized PnL, PM команды
7. Вложенные строки "Трейдер": ФИО, Капитал, Unrealized PnL, Realized PnL, Доля (% в команде)
8. Клик по трейдеру открывает список открытых сделок снизу
9. Клик по сделке открывает карточку сделки с графиком
10. Сортировка, поиск по имени, постраничная навигация (>50 строк)
11. Цвета: зелёный-прибыль, красный-убыток, серый-нет данных
12. Пустые состояния: "Нет данных за выбранный период"

**Подвкладка "История сделок":**
13. Фильтры: Трейдер, Старший трейдер, Период, Инструмент, Направление, Статус, Результат. Кнопки Сбросить и Применить
14. Таблица сделок: Трейдер, Инструмент, Позиция, Объём, Плечо, Режим, Цена входа, Стоимость, SL, TP, Unrealized PnL, Realized PnL, Дата входа, Дата обновления, Действие
15. Кнопка "Подробности" -- модал/страница с графиком (маркеры входа/выхода), параметры сделки, текст дневника, лог изменений

## Раздел 4: Тим Лид -- Риск-менеджмент
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#04_tl_risk
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/risk-management.html

### Требования из ТЗ для проверки:
1. Два таба внутри: Команда и Политики
2. Верхняя сводка: "Суммарный риск сейчас" ($ и %), счётчики "Предупреждения за период" / "Принудительные закрытия за период" (кликабельные)
3. Статусы ограничений -- мини-чипы с числами: "На тестировании", "Запрет торговли", "Запрет по монетам"
4. Фильтры сводки: Период, Старший трейдер, Политика, Монета/рынок
5. Таблица "Команда" колонки: Трейдер, Политика РМ (дропдаун!), Риск сейчас (до SL), Остаток риска на период (прогресс-бар), Предупреждения (число, клик), Принудительные закрытия (число, клик), Открыто позиций (число, клик)
6. Дропдаун "Политика РМ" -- мгновенное назначение
7. Прогресс-бар "Остаток риска" с цветовыми порогами
8. Сортировка, пагинация, фильтры
9. Автообновление 15-30 сек без мигания
10. Экспорт CSV
11. Журнал действий РМ -- ссылка под таблицей

## Раздел 5: Тим Лид -- Орг.структура
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#05_tl_org
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/org-structure.html

### Требования из ТЗ для проверки:
1. Переключатель вверху: "Список" / "Орг.структура (дерево)"
2. Тим-лид -- корень (верх/центр)
3. Инвесторы -- на уровне Тим-лида, горизонтальные пунктирные линии (связи, не подчинение)
4. Ниже -- пары [Старший трейдер] -- [Риск-менеджер] горизонтально
5. Под каждой парой -- вертикальный список "Команда трейдеров"
6. Узел "Без пары" для неназначенных трейдеров
7. Раскрытие/сворачивание уровней: ТЛ -> Пары -> Команда
8. Кнопки "Развернуть все / Свернуть все"
9. Поиск по имени/нику/email -- автораскрытие веток с совпадениями
10. Клик по трейдеру -- попап "Аналитика трейдера"
11. Скролл при большом количестве узлов
12. Линии: вертикальные=подчинение, горизонтальные пунктирные=связи
13. Пустые состояния: "Нет участников" для пустых пар, "Ничего не найдено" при поиске

## Раздел 7: Трейдер -- Дневник
**Эталон:** https://ipostny.github.io/trigonum-design-mockups/reference.html#07_trader_journal
**Мокап:** https://ipostny.github.io/trigonum-design-mockups/trader-workspace.html

### Требования из ТЗ для проверки:
**Подвкладка "Торговая система":**
1. Встроенный текстовый редактор (заголовки, списки, ссылки)
2. Карточка загруженного файла DOC/DOCX/PDF с кнопкой "Скачать"
3. Если есть текст и файл: сверху текст, ниже карточка файла
4. Строка "Последнее обновление: ДД.ММ.ГГ ЧЧ:ММ, Автор"
5. Кнопки для трейдера: Редактировать, Загрузить/Заменить файл, Сохранить, Отмена
6. Для ТЛ/СТ/РМ: только просмотр и Скачать

**Подвкладка "История сделок":**
7. Фильтры: Период, Инструмент, Направление (LONG/SHORT), Статус, Плечо
8. Таблица: Дата/время, Инструмент, Направление, Объём, Цена входа, Текущая/Цена выхода, P&L $, P&L %, Тейк-профит, Стоп-лосс, Причины входа
9. Клик по строке -- попап "Детали сделки" с редактируемыми полями: Причина входа, Комментарий, SL, TP, RR, Риск на сделку
10. P&L подсветка: прибыль зелёным, убыток красным
11. Пустое состояние: "Нет сделок за выбранный период"
